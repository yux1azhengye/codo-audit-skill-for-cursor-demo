# 代码审计终报模板

终报文件名为 `audit-report.md`，置于用户指定的报告目录（默认 `./audit-report/`）。

## 文档结构

```markdown
# 代码安全审计报告

**项目路径**：<目标路径>
**报告时间**：<日期>
**扫描范围**：<全库 / 仅变更>
**主语言/技术栈**：<来自 Recon>

---

## 1. 执行摘要

- 确认漏洞总数：<N>
- 按类型分布：<SQL / RCE / SSRF / 认证 / 文件 / 逻辑 / 其他>
- 按严重程度（若使用）：Critical / High / Medium / Low 数量

---

## 2. 已确认漏洞清单

（以下每条必须包含下述四要素，缺一不可）

### 2.1 [漏洞类型] - [简短标题]

- **API / 入口 URL**：`METHOD /path` 或具体 URL
- **Source → Sink**：从哪一参数/Header/Body 到哪一文件:行或调用
- **PoC / 复现步骤**：可复现的请求示例或步骤（HTTP 请求可贴 curl 或关键参数）
- **修复建议**：具体代码级或配置级修复方案
- **备注**：（可选）已动态验证 / 可组合攻击链 / 严重程度

（重复 2.1 直至所有确认漏洞写完）

---

## 3. 附录（可选）

- 已排除的误报类型说明
- 待确认项（未进终报的条目及原因）
- 攻击链说明（若存在多漏洞组合）
```

## 单条发现硬性格式

| 字段         | 必填 | 说明 |
|--------------|------|------|
| API/入口 URL | 是   | 可定位到接口的路径或 URL |
| Source→Sink  | 是   | 数据流起止，便于开发定位 |
| PoC/复现步骤 | 是   | 无 PoC 的不得进入终报 |
| 修复建议     | 是   | 具体可执行建议 |

未满足上述格式的条目不得写入 `audit-report.md`。

# Round 1 专项：认证 / 鉴权 / 越权

## 目标

识别认证绕过、水平/垂直越权（IDOR）、权限校验缺失或可篡改的接口与数据流。

## 策略

1. 从 Recon 的 `auth`、`routes` 出发，列出所有需要“登录/权限”的路由及校验方式。
2. 枚举接收 `user_id`、`id`、`role`、`is_admin`、`tenant_id` 等参数的 API，追踪参数在 Service 层的使用：是否用于鉴权、是否可与当前用户不一致（IDOR）。
3. 检查权限校验是否在“入口层”统一做（如 Filter/中间件）且无绕过；是否存在仅前端校验、后端未校验的接口。
4. 检查 Token/Session 的生成、校验、过期与注销逻辑是否可绕过或重放。

## 输出（写入 round1-findings）

每条：**API/路由**、**Source（参数/Header）**、**问题类型（绕过/IDOR/校验缺失）**、**数据流简述**。二轮补 PoC（如替换 user_id 访问他人数据）。

## 逻辑强化（参考陌陌安全思路）

以“理解业务的高级渗透测试专家”视角：重点审视参数篡改、状态机绕过、越权访问；利用变量名语义（`user_id`、`role`、`skip_auth`）定位敏感接口，查其是否参与鉴权、是否可被篡改。
